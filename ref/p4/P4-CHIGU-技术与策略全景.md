## 一、系统定位与设计目标

P4-CHIGU 是一个**持仓风险监控系统**，服务于 A 股个人投资者的日常持仓管理。核心能力：

- **盘前**：基于 T-1 收盘数据，对持仓进行多维度风险评分，给出减仓/止损建议
- **盘中**：每 3 分钟监控实时行情，捕捉急跌、放量异常、关键位跌破等盘中事件
- **盘后**：汇总当日表现、LLM 点评、波段位置，推送到 Telegram

**不做的事**：选股、自动交易、回测优化。系统只回答一个问题——**现在持有的票，风险有多大，该不该减仓**。

## 三、核心量化策略：三层风险评分

### 3.1 总分公式

```
risk_total = W_ml × risk_prob + W_rules × rule_score + W_portfolio × portfolio_adj
```

| 层 | 默认权重 | 输入 | 输出范围 |
|----|---------|------|---------|
| ML 模型 | 60% | 30 个日线特征 | 0.0 ~ 1.0 |
| 规则引擎 | 30% | 日线 + 资金流 + 盘口 | 0.0 ~ 0.30 (硬顶) |
| 组合风险 | 10% | 全持仓交叉分析 | -0.05 ~ +0.10 |

**风险等级划分（受市场状态调节）**：

| 状态 | green | yellow | orange | red |
|------|-------|--------|--------|-----|
| 牛市/中性 | < 0.18 | < 0.30 | < 0.45 | ≥ 0.45 |
| 熊市 (×0.80) | < 0.144 | < 0.240 | < 0.360 | ≥ 0.360 |

**动态权重降级**：当 ML 输出方差 < 0.01（模型"摆烂"，所有票分数差不多），自动切换为 ML=0%, Rules=65%, Portfolio=35%。

**硬底线覆盖**：
- 浮亏 > 15% → 至少 orange
- 区间回撤 > 20% → 至少 orange
- 卖出紧迫度 ≥ 50 但 green → 上调为 yellow

### 3.2 ML 模型层

**算法**：LightGBM 5 分类器

| 参数 | 值 |
|------|-----|
| n_estimators | 1000 |
| learning_rate | 0.05 |
| max_depth | 5 |
| num_leaves | 31 |
| min_child_samples | 40 |
| subsample | 0.8 |
| colsample_bytree | 0.7 |
| reg_alpha / reg_lambda | 0.1 / 0.1 |
| class_weight | {0: 2.0, 1: 0.5, 2: 3.0} |

**5 类风险映射**：

| 类别 | 语义 | risk_weight | class_weight |
|------|------|-------------|-------------|
| 0 | 无风险 | 0.00 | 2.0 (强调) |
| 1 | 低风险 | 0.15 | 1.0 |
| 2 | 中风险 | 0.40 | 0.5 (压低) |
| 3 | 高风险 | 0.75 | 1.0 |
| 4 | 极高风险 | 1.00 | 3.0 (强调) |

**risk_prob 计算**：

```
risk_prob = Σ (P(class_i) × risk_weight_i)
```

例：P = [0.1, 0.3, 0.3, 0.2, 0.1] → risk_prob = 0×0.1 + 0.15×0.3 + 0.4×0.3 + 0.75×0.2 + 1.0×0.1 = 0.415

**训练池筛选条件**：
- 市值 50 亿 ~ 1000 亿
- 日均成交额 ≥ 3000 万
- 上市天数 ≥ 250

### 3.4 规则引擎层

8 条启用规则，8 条待修复（已禁用）。规则分最终 capped at 0.30。

**启用规则**：

| 规则 | 权重 | 严重度 | 触发条件 |
|------|------|--------|---------|
| 均线破位 | 0.12 | high | MA20偏离<0 且 MA10斜率<0；首次破位=high，持续在下=low |
| 硬回撤兜底 | 0.12 | high | 从入场/20日高点回撤 > 7% |
| 下跌波动偏高 | 0.10 | medium | 近10日下跌日std / 上涨日std > 1.5 |
| 连续缩量阴跌 | 0.10 | medium | 连续 ≥ 3 日下跌且缩量 |
| 突破失败 | 0.10 | high | 近3日高点接近20日高×0.98，收盘未站上，量能不足(<5日均量80%) |
| SuperTrend翻转 | 0.10 | high | SuperTrend(10, 3.0×ATR) 方向由上转下 |
| MACD动量衰减 | 0.08 | medium | MACD柱 > 0 但连续 ≥ 3 日缩减 |
| 流动性枯竭 | 0.08 | medium | 20日换手率分位数 < 10% |

**禁用规则**（待数据源修复）：
波动率飙升、量价背离、大单持续流出、尾盘杀跌、向下跳空、持续弱于行业、北向资金卖出、放量滞涨

### 3.5 组合风险层

| 检查项 | 阈值 | 触发调整 |
|--------|------|---------|
| 行业集中度 | 单行业占比 > 40% | +0.04 |
| 持仓相关性 | 20日收益两两相关系数均值 > 0.60 | +0.03 |
| 整体风险暴露 | 全持仓 risk_prob 均值 > 0.25 | +0.03 |

最终 portfolio_adj 范围：-0.05 ~ +0.10

## 四、卖出紧迫度评分（独立维度）

与 risk_total 并行的独立评分系统，不参与风险等级计算，直接用于仓位建议。

### 4.1 五个子维度

| 维度 | 上限 | 信号来源 |
|------|------|---------|
| momentum (动量) | 30 | MACD衰减(+15), SuperTrend翻转(+15) |
| structure (结构) | 25 | 均线破位(首次+12/持续+5), 硬回撤(梯度+5~20) |
| volume (量能) | 25 | 量价背离(+12), 放量滞涨(+8), 大单流出(+5) |
| stop (止损) | 20 | Chandelier突破(+20), 距止损<2%(+10), 硬回撤梯度(+5~20) |
| pnl (浮盈亏) | 20 | 浮亏5%~20%梯度加分 |

### 4.2 硬回撤梯度评分

| 回撤幅度 | structure 加分 | stop 加分 |
|----------|---------------|----------|
| ≥ 15% | +20 | +20 |
| ≥ 10% | +15 | +15 |
| ≥ 7% | +10 | +10 |
| < 7% | +5 | +5 |

### 4.3 紧迫度等级

| 分数 | 等级 | 含义 |
|------|------|------|
| ≥ 75 | urgent | 建议立即减仓 |
| 50-74 | reduce | 建议减仓 |
| 25-49 | watch | 关注观察 |
| < 25 | hold | 正常持有 |

**新仓保护**：持仓 ≤ 3 天，信号放大 ×1.3（新仓出问题更要快跑）。

### 4.4 仓位建议矩阵

risk_level × urgency_level → 建议仓位比例：

| | hold | watch | reduce | urgent |
|---|---|---|---|---|
| green | 100% | 80% | 60% | 40% |
| yellow | 80% | 60% | 40% | 20% |
| orange | 60% | 40% | 20% | 0% |
| red | 40% | 20% | 0% | 0% |

## 五、波段引擎

独立于风险评分，用于判断当前价位在历史区间中的位置，辅助高抛低吸决策。

### 5.1 综合评分

```
band_position = valuation_score × 0.40 + technical_score × 0.60
```

### 5.2 估值评分 (0-100)

- ETF：固定 50（无 PE 数据）
- 个股：PE_TTM 在近 750 个交易日（约 3 年）中的百分位 × 90 + 5

### 5.3 技术评分 (0-100)

三个子指标等权：
1. **布林位置**：(close - lower) / (upper - lower) × 100，Bollinger(20, 2σ)
2. **RSI(14)**：直接使用 0-100 原始值
3. **MA20 偏离**：(ma20_dev + 0.10) / 0.20 × 100，即 -10%→0, +10%→100

### 5.4 波段等级

| 区间 | 等级 | 含义 |
|------|------|------|
| ≤ 15 | oversold | 超卖 |
| 16-35 | low | 偏低 |
| 36-65 | neutral | 中性 |
| 66-85 | high | 偏高 |
| > 85 | overbought | 超买 |

## 六、市场状态检测

### 6.1 市场 Regime 判断

```python
detect_regime(mkt_up_ratio, hs300_5d_return, hs300_ma20_bias, consecutive_down_days)
→ "bear" | "neutral" | "bull"
```

| 状态 | 触发条件 |
|------|---------|
| bear | 上涨比例 < 35%；或上涨比例 < 40% 且沪深300近5日跌 > 2%；或连续下跌 ≥ 3 天 |
| bull | 上涨比例 > 60% 且沪深300偏离MA20 > 2% |
| neutral | 其他 |

熊市时风险阈值 ×0.80，即更容易触发高风险等级。

### 6.2 盘中市场温度（Market Pulse）

4 维度打分（满分 100）：

| 维度 | 权重 | 计算 |
|------|------|------|
| 日内涨跌 | 30 分 | 15 + 涨跌幅% × 7.5 |
| 1小时趋势 | 30 分 | 15 + 小时涨跌% × 10 |
| 日内波动 | 20 分 | 20 - 振幅% × 6.7 |
| 距日低位置 | 20 分 | (close-day_low)/(day_high-day_low) × 20 |

**温度等级 → 风险灵敏度系数**：

| 温度 | 分数 | 系数 | 效果 |
|------|------|------|------|
| danger | < 25 | 0.80 | 盘中规则阈值收紧 20% |
| weak | 25-49 | 0.90 | 收紧 10% |
| neutral | 50-74 | 1.00 | 基准 |
| healthy | ≥ 75 | 1.20 | 放松 20% |

## 七、盘中监控（9 条实时规则）

每 3 分钟触发一轮，数据来源为 pytdx 实时 1 分钟线 + 五档行情。

| # | 规则 | 等级 | 触发条件 |
|---|------|------|---------|
| 1 | 放量下杀 | RED | 30分钟跌幅 < -2% 且量比 > 2.0 |
| 2 | 急速下跌 | RED | 5分钟跌幅 < -1.5% |
| 3 | 关键位跌破 | ORANGE | 创日内新低 |
| 4 | VWAP偏离恶化 | ORANGE | 低于 VWAP ≥ 30分钟 且偏离 < -1.0% |
| 5 | 反弹无力 | ORANGE | 跌幅 > 2% 后反弹不足跌幅 50% |
| 6 | 量价异常 | YELLOW | 最近成交量 > 3倍均量 且涨跌幅 < 0.5% |
| 7 | 逼近止损 | YELLOW | 下行趋势中距日低 < 0.5% |
| 8 | 盘口失衡 | YELLOW | 卖五档总量 / 买五档总量 > 3.0 |
| 9 | 跌破10日低点 | ORANGE | 收盘价 < 10日最低价 |

**告警冷却机制**：
- 同一股票同类型告警：30 分钟内不重复
- 同一股票任意告警：10 分钟内不重复
- 全局：每小时最多 8 条

## 九、数据管道

### 9.1 数据源优先级

```
Tushare Pro (主力) → AkShare (校验/备用) → BaoStock (最后兜底)
pytdx (仅盘中实时行情，不参与日线更新)
```

### 9.2 Pipeline 三步流程

```bash
python scripts/pipeline.py pull      # 步骤1：拉取日线/资金流/基本面
python scripts/pipeline.py features  # 步骤2：计算 30+ 日线特征
python scripts/pipeline.py train     # 步骤3：训练 LightGBM 模型
python scripts/pipeline.py all       # 全部执行
```

**步骤1 Pull**：
- 交易日历（AkShare）
- 持仓 + 训练池日线（Tushare → AkShare 备用，每只股票最多重试 3 次）
- 指数日线（沪深300、中证1000）
- 资金流数据（moneyflow_daily）
- 基本面数据（PE/PB/PS/换手率/市值）

**步骤2 Features**：
- 对比 features_daily 与 market_bars_daily 的最新日期
- 只更新有新数据的股票，批量 200
- 注入基本面指标（PE/PB/PS/换手率/ROE/营收增速）
- 注入指数 alpha（沪深300、中证1000）

**步骤3 Train**：
- 验证集 = 最近 90 天
- 输出 `models/risk_latest.pkl`
- 注册到 model_registry 表

### 9.3 特征工程详情（70+ 列）

除 ML 使用的 30 个特征外，features_daily 还包含：

**均线系列**：MA5/10/20/60 偏离、MA10 斜率、MA5 是否在 MA20 上方、MA20 是否在 MA60 上方

**动量扩展**：10日/20日/60日收益率、动量加速度 (ret_5d - ret_5d[-5])

**技术指标全集**：RSI(6/14) + 超买超卖标记、MACD(DIF/DEA/HIST) + 金叉死叉、KDJ(K/D/J) + 超买超卖、布林带位置和宽度

**价格结构**：距20日/60日高低点距离、收盘在区间中的位置、开盘跳空、日内位置

**估值指标**：PE_TTM、PB、PS_TTM、ROE_TTM、营收增速YoY、距报告期天数

**市场关联**：Beta(60d)、特质波动率(20d)、Amihud 非流动性

## 十一、评估体系

### 11.1 分离度（核心评估指标）

衡量"红色预警的票是否真的比绿色票跌得多"。

```
分离度 = 红色组平均5日最大回撤 - 绿色组平均5日最大回撤
```

分离度越大说明模型区分能力越强。低于 2.0 时系统发出"考虑重训练"的警告。

### 11.2 回填验证

- T+5 回填：每条风险评估记录，5 天后回填实际最大回撤
- 卖出信号回填：每条卖出紧迫度记录，回填后续实际表现
- 盘中告警回填：每条盘中告警，回填后续跌幅

### 11.3 周报与月报

- **周报**：分桶统计（red/orange/yellow/green 各桶的平均 5 日 DD）、规则命中率、分离度趋势
- **月报**：整体评分、模型表现、策略建议

### 12.2 调用场景

|------|------|------|------|
